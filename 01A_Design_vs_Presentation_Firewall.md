# 01A_Design_vs_Presentation_Firewall

## Purpose

Prevent presentation logic from overrunning design logic.

This note enforces the partition defined in [[Chapter2_Obsidian/01_Governing_Memo]] during writing.

---

## Rule

Design decides what enters.  
Presentation does not justify inclusion.

---

## Immediate Test (apply while writing)

For any paragraph, section, or note:

1. Is this CORE?
2. If not, is it strictly required from the RING?
3. If not, it is OUTER FIELD → do not include.

If unclear, default to exclusion.

---

## CORE Enforcement

Only the following are admissible as primary content:

- capacity utilization problem
- Shaikh–Okishio–Vidal articulation
- center vs Chile comparison
- mechanisms required to interpret that comparison

If a paragraph does not strengthen one of these, it is not CORE.

---

## Presentation Triggers (danger signals)

If you write a sentence that:

- introduces “background”
- summarizes literature without decision role
- expands context “for clarity”
- explains what the section will do
- adds examples not required for inference
- restates a point already made

→ STOP

That is presentation drift.

---

## Correction Protocol

When drift is detected:

1. Do not expand further  
2. Identify which CORE element this was supposed to support  
3. Either:
   - rewrite it to serve that function  
   - or delete it  

No patching with more prose.

---

## RING Discipline

RING elements are allowed only when they:

- orient the comparison (imperial hierarchy, dependency)
- anchor the field minimally (Zavaleta)

They must:
- appear briefly  
- not generate subsections  
- not introduce new analytical fronts  

If they expand, they become OUTER FIELD.

---

## OUTER FIELD Lock

The following are blocked from entry:

- profitability system
- full world-historical reconstruction
- dataset architecture expansion
- ML / methodological extensions

If they appear during writing:

→ move to a separate note  
→ do not keep in draft text  

---

## One-Burden Rule

Each paragraph must perform **one function only**:

- define  
- specify mechanism  
- compare  
- interpret  

If a paragraph does more than one, split or cut.

---

## Hard Stop Rule

If a section grows but does not make the capacity-utilization argument clearer:

→ stop writing  
→ return to [[Chapter2_Obsidian/01_Governing_Memo]]  
→ re-anchor before continuing  

---

## Final Check

Before accepting any section:

- Can this be removed without weakening the core argument?  
  → If yes, remove it  

- Does this belong to a future chapter or appendix?  
  → If yes, move it
